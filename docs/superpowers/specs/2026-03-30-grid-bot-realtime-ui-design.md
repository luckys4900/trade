# グリッドボット リアルタイムUI設計

**日付:** 2026-03-30
**バージョン:** 1.0
**ステータス:** デザイン承認待ち

---

## 1. 概要

グリッドトレーディングボットに対して、**リアルタイムのチャート表示とエントリータイミング可視化のWeb UI** を追加する。

ユーザーが現在価格 → グリッドレベルまでの距離、準備度、エビデンス（RSI/ATR/時間）を**ビジュアルに一覧できる**ことで、エントリー機会の判断を高速化・最適化する。

### 主要な目標
1. ✅ **価格レベルの可視化** — 現在価格と次のエントリーレベルをチャートに表示
2. ✅ **エントリー準備度** — READY/WARN/FAR の3段階ゲージで準備状況を表示
3. ✅ **エビデンス表示** — 価格差額、時間推定、RSI/ATR などを数値で明記
4. ✅ **リアルタイム監視** — 1分ごとに自動更新、グリッド状態を常に反映

## 2. アーキテクチャ

### 2.1 全体構成

```
┌─────────────────────────────────────────┐
│  GridBot（既存の自動取引エンジン）       │
│  - grid_bot.py                          │
│  - grid_manager.py                      │
└────────────────┬────────────────────────┘
                 │
            (データ取得)
                 │
┌────────────────▼────────────────────────┐
│  リアルタイムUI（新規）                  │
│  - ui_server.py（Streamlit）            │
│  - chart_builder.py（Plotly チャート）  │
│  - state_manager.py（UI状態管理）       │
└────────────────┬────────────────────────┘
                 │
         (ブラウザで表示)
                 │
        ┌────────▼────────┐
        │ユーザー監視画面  │
        │http://localhost:8501
        └─────────────────┘
```

### 2.2 データフロー

1. **GridBot が稼働** — グリッド注文を配置・管理
2. **UI サーバー が定期取得**（1分間隔）:
   - 現在価格（API から）
   - グリッドレベル（GridManager から）
   - インジケーター（RSI、ATR）
   - グリッド状態（open_orders, filled_levels）
3. **UI が計算**:
   - 次のエントリーレベルを特定
   - 価格差額 → パーセンテージ変換
   - 時間推定（ATR ベース）
   - READY/WARN/FAR の判定
4. **ブラウザで表示** — Plotly チャート + 右パネル情報

## 7. エビデンス計算の詳細

### 7.1 READY/WARN/FAR の判定

```python
distance_pct = (next_level - current_price) / current_price * 100

if abs(distance_pct) <= 1.0:
    state = "READY"  # 緑
elif abs(distance_pct) <= 5.0:
    state = "WARN"   # 黄
else:
    state = "FAR"    # 赤
```

### 7.2 時間推定

```
推定時間 = 距離（ドル） / (ATR / 24)
         ≈ abs(next_level - current_price) / (atr_value / 24)

例: 距離 $234, ATR $850
   推定時間 = 234 / (850/24) ≈ 6.6 時間
```

### 7.3 RSI 解釈

```
RSI < 30: 過度に売られている（警告）
30 <= RSI <= 70: 中立（理想的）
RSI > 70: 過度に買われている（警告）
```

## 9. ユースケース・検証

### ユースケース 1: 価格が次レベルに接近
```
シナリオ: BTC 価格 $67,234 → 次買いレベル $67,000 (SL)

表示（右パネル）:
- 準備度ゲージ: [████████░░] 95% 接近 (WARN)
- SL (買いレベル): $67,000 (-0.35% = $234)
- TP (売りレベル): $68,000 (+1.15% = +$766)
- R/R比: 1:3.27（リスク $234 vs リワード $766）
- 推定到達: SL まで 4時間、TP まで 12時間
- RSI: 45（中立 → エントリー環境良好）

表示（チャート上）:
- 赤い破線: 次の買いレベル (SL)
- 緑の破線: 次の売りレベル (TP)
- 黒い太線: 現在価格
- これらが視覚的に一目瞭然

判定: ユーザーは「あと 0.35% で SL に到達、その後 TP まで利益狙い。R/R比 1:3.27 は優良」と判断可能
```

### ユースケース 2: グリッド再計算の準備
```
シナリオ: 価格が現在グリッドから ±18% 乖離

表示:
- ⚠️ グリッド再計算まで: ±18% 残り
- 警告色で強調

判定: ユーザーは「グリッドが更新されるな」と認識
```

### ユースケース 3: リスク警告
```
シナリオ: RSI > 75（過度に買われている）

表示:
- RSI: 78 (警告色・黄)
- 注釈: 「⚠️ 強い上昇トレンド中」

判定: ユーザーは「エントリーは慎重に」と判断
```
