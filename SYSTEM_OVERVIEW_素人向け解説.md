## 🤖 **5つのロボットの役割**

### **1️⃣ Daily SMA Trader（毎日分析ロボット）**

**何をしているか**:
```
毎日のビットコイン価格をチェック
     ↓
「今のトレンドは上向き？下向き？」を判定
     ↓
上向き → BTC買い
下向き → BTC売却
```

**具体例**:
```
5月1日（水）
  BTC価格: $65,000
  → トレンド: 上向き
  → ロボット判断: 買い！
  → 自動で $130を使ってBTCを買い
  
5月5日（日）
  BTC価格: $67,000（上がった！）
  → トレンド: 下向きに転換
  → ロボット判断: 売り！
  → 自動で買ったBTCを売却
  → 利益: $200 - $130 = $70 ✅
```

**使用データ**:
- 過去365日の毎日のBTC価格
- 移動平均線（5日と17日）という線を計算
- 2本の線の交差点で「買い」「売り」を判定

**成績**:
- バックテスト Sharpe比: **4.16**（非常に良い）
- 統計的有意性: p=0.16（完全には証明されていないが有望）

---

### **2️⃣ Whale Monitor（くじら監視ロボット）**

**何をしているか**:
```
大口投資家（「くじら」と呼ぶ）の動きをリアルタイム監視
     ↓
「あ、くじらが買い始めた！」と検知
     ↓
あなたのロボットも一緒に買う（真似する）
```

**具体例**:
```
大口投資家A（資金: $500万）
  15分前: BTC買い → $1,000,000投入
  ↓（ロボットが検知）
  
あなたのロボット:
  「あ、くじらが動いた！」
  ↓
  あなたの $190の中から $50 を使って買い

大口投資家A:
  1時間後: 利益確定で売却
  ↓（価格が上がったから）
  
あなたのロボット:
  自動で売却 → 利益 ✅
```

**監視対象**:
- 3人の大口投資家の動き
- 彼らのポートフォリオ（保有資産）
- 彼らの過去の成績（勝率など）

**期待値追跡**:
- 「くじらが買った後、平均何%上がるか」を計算
- 実際の利益と理論値を比較
- 30日ごとに本当に稼げているかチェック

---

### **3️⃣ Macro Filter（相場環境判定ロボット）**

**何をしているか**:
```
相場全体の「危険度」をチェック
     ↓
危険な時期 → 売買を一時停止
安全な時期 → 通常通り売買
```

**具体例**:
```
【安全な環境】
  ボラティリティ（価格の変動幅）: 低い
  経済ニュース: 平穏
  → 売買OK ✅

【危険な環境】
  ボラティリティ: 異常に高い（暴落の可能性）
  経済ニュース: FRB金利決定、戦争勃発など
  → 売買一時停止 ⚠️
  → あなたの資金を守る
```

**監視項目**:
- BTC価格の変動の大きさ
- 経済指標（雇用統計、インフレ率など）
- 市場全体の取引量
- 重要ニュースのタイミング

---

### **4️⃣ Kronos Predictor（逆張り予測ロボット）**

**何をしているか**:
```
相場が「過度に買われている」or「過度に売られている」を検知
     ↓
その反対の方向に賭ける（逆張り）
```

**具体例**:
```
【過度に買われている場合】
  みんながパニックで買っている状態
  （実は割高なのに買い続けている）
  ↓
  Kronos: 「そろそろ売られるな」と予測
  ↓
  逆張り: 売りポジションを建てる
  ↓
  予測通り下がる → 利益 ✅

【過度に売られている場合】
  みんながパニックで売っている状態
  （実は割安なのに売り続けている）
  ↓
  Kronos: 「そろそろ買い戻されるな」と予測
  ↓
  逆張り: 買いポジションを建てる
  ↓
  予測通り上がる → 利益 ✅
```

**使用技術**:
- RSI（相対力指数）: 過度に買われた/売られたを判定
- MACD: トレンド転換を検知
- 機械学習（AI）: 過去パターンから学ぶ

## 💰 **期待値（EV）をどう追跡しているか**

### **概念図**

```
期待値とは:
「1回の取引で、平均いくら儲かるか」

【例】
100回取引した結果:
  勝ち: 60回 × 平均 +$1.50 = +$90
  負け: 40回 × 平均 -$0.80 = -$32
  
期待値 = ($90 - $32) / 100回 = +$0.58/回
```

### **実際の追跡方法**

```
【リアルタイム追跡】
毎回の取引:
  ├─ エントリー（買い/売りの開始）時刻・価格
  ├─ エグジット（利確/損切）時刻・価格
  ├─ 利益/損失 計算
  └─ ログに記録

【日単位の集計】
  ├─ 今日の勝ち取引: 3回
  ├─ 今日の負け取引: 1回
  ├─ 今日の合計利益: +$2.45
  └─ 今日の期待値: +$0.61/回

【週/月単位での検証】
  ├─ 「理論では +0.5%/回」
  ├─ 「実際は +0.61%/回」
  ├─ 「理論通り動いてるか？」確認
  └─ 必要に応じてパラメータ調整
```

## ⚠️ **期待値を生むメカニズム｜3つの要素**

### **要素1: トレンド追従（Daily SMA）**

```
基本原理: 「傾向は続く」

    価格
    ▲
    │     ╱╱╱ ← トレンド上向き
    │   ╱╱
    │ ╱╱ ← ここで買う
    │╱
    └────────→ 時間

買ったタイミング: トレンドが「下向き→上向き」に転換した直後
売るタイミング: トレンドが「上向き→下向き」に転換した直後

期待値: 上向きトレンド中の上昇幅の一部を獲得
```

### **要素2: スマートマネーの後追い（Whale Monitor）**

```
基本原理: 「大口が儲かるトレードなら、僕たちもそこそこ儲かる」

大口投資家の軌跡:
    ↑ 資金投入
    ├─ 買い値: $65,000
    │
    ├─ トレンド上昇
    │
    ├─ 利確: $67,000（+3%）
    ↓

小口（あなた）:
    ├─ 大口の買いを検知（15分遅延）
    ├─ 買い値: $65,200（既に少し上がってる）
    │
    ├─ トレンド上昇
    │
    ├─ 売り: $66,800（+2.5%）
    ↓

期待値: 
  大口が買うと、その後平均+3%
  → 小口は遅延するから平均+2%は期待できる
```

### **要素3: リスク管理による「ドローダウン最小化」**

```
戦略Aの期待値: +0.5%/回
但し最悪ケース: -15% の損失あり

戦略Bの期待値: +0.5%/回
同じ期待値だが、Macro Filterで危険時期を避けるので
最悪ケース: -5% 程度

→ 同じ期待値でも、リスク調整後ロボットBが有利

Sharpe比 = 期待値 / リスク の大きさ

Macro Filterがあることで Sharpe比がアップ ✅
```

## 🔍 **なぜバックテストされた4つは却下されたか**

### **FR Carry Trade が失敗した理由**

```
【理論値】
「8時間ごとに資金調達料が支払われる」
→ 毎月 +0.8% くらい儲かるはず

【実測値】
パックテスト実行
→ 実は -0.09%/月

【原因】
取引手数料: 0.17% (往復)
  ↓
いくら資金調達料をもらっても、
手数料で吸収されてしまう

【教訓】
「手数料 > 獲得利益」なら絶対に儲からない
理論だけでなく、コストも含めて計算すべき
```

### **Inside Bar Pattern が失敗した理由**

```
【理論値】
「内側にいるバーからのブレイクアウトは、
  80%の確率でトレンド継続」
（一般的な相場論）

【実測値】
BTC 4時間足で362個のパターン検出
  ↓
統計的には「ランダム」と区別つかず
p=0.60 (p<0.05が目安なので、完全に失敗)

【原因】
一般的な相場論は、為替相場や株式では有効だが、
BTC は ボラティリティ が高く、
パターン認識が通用しない

【教訓】
「教科書通り」では通用しない
BTCは独特のダイナミクスがある
```
