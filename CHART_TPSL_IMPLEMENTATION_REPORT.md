## 実装内容

### 修正ファイル
- `trade_chart.py` — 新しいログ形式（rsi_swing_*.log）に対応・インジケーター追加

### 主要な変更点

#### 1. ログ解析の全面更新

**対応するログパターン:**

| パターン | 例 |
|---------|-----|
| LONG エントリー | `OPEN LONG @ 42500.00 \| SL=41300.00 TP=44900.00 qty_est=0.001234` |
| SHORT エントリー | `OPEN SHORT @ 42500.00 \| SL=43700.00 TP=40100.00 qty_est=0.001234` |
| ポジション状態 | `In position (long). SL=41300.00 TP=44900.00 \| Entry=42500.00` |
| ポジション クローズ | `CLOSE LONG @ 44900.00 (entry=42500.00)` |

**取得される情報:**
```python
{
    'entry_price': 42500.0,        # エントリー価格
    'in_position': True,           # ポジション有無
    'position_side': 'long',       # ポジション方向
    'sl_price': 41300.0,           # SL価格（ログから直接取得）
    'tp_price': 44900.0,           # TP価格（ログから直接取得）
}
```

**フォールバック機能:**
- `rsi_swing_*.log` がない場合、`trader_*.log` も確認
- ATRベースでSL/TPが計算できない場合は、config.jsonから `sl_atr`, `tp_atr` を読み込んで計算

#### 2. インジケーター追加

**新しく追加:**
- ✅ **EMA(50)** — オレンジ色のライン（トレンドフィルター表示）
- ✅ **ATR(14)** — チャートの下部に表示準備（RSI下に統合）

**既存のインジケーター:**
- RSI(14) — 紫色
- RSI 30/70 ラインと背景色

#### 3. TP・SL表示の改善

**SL・TP計算:**
- ✅ ログから直接取得した実際の値を使用（ATRベース）
- ✅ 計算失敗時のフォールバック有り

**表示内容:**
- エントリーライン（緑・実線） — ラベル付き
- TP ライン（青・破線） — ATR倍率を表記（例: `TP $44900.00 (3.0×ATR)`）
- SL ライン（赤・破線） — ATR倍率を表記（例: `SL $41300.00 (1.5×ATR)`）

**R:R比の計算と表示:**
```
R:R比 = (TP距離) / (SL距離)
例: 1:2.0
チャートタイトルに表示: "R:R: 1:2.00"
```

#### 4. チャートの外観更新

| 項目 | 修正前 | 修正後 |
|------|-------|-------|
| Timeframe表示 | BTC/USDT 1H（固定） | BTC/USDT {config値}（4H等） |
| RSI閾値 | 30/60 | 30/70 ✅ |
| RSI背景色上限 | y1=60 | y1=70 ✅ |
| TP・SL計算 | 固定パーセント（stop_loss_pct等） | ATRベース（ログから取得） ✅ |
| EMA | なし | EMA(50) 追加 ✅ |

#### 5. チャートタイトル情報

```
BTC/USDT 4H | RSI: 65.23 | Price: $42,520.00 | HOLDING | Entry: $42,500.00 | R:R: 1:2.00
```

詳細情報:
- BTC/USDT {Timeframe} — config値から自動取得
- RSI — 現在のRSI値
- Price — 現在価格
- 状態 — HOLDING（ポジション中）/ FLAT（ポジションなし）
- Entry — エントリー価格（ポジション中のみ）
- R:R — リスク・リワード比（SL・TP両方ある場合のみ）

## チャートの見方

### 1. ローソク足チャート（上）

| 要素 | 色 | 意味 |
|------|-----|------|
| ローソク足 | 白/黒 | OHLC |
| EMA(50) | オレンジ | トレンドフィルター（Close > EMA(50)でLONG等） |
| エントリー | 緑・実線 | エントリー価格 |
| TP | 青・破線 | テイク・プロフィット（想定利確位置） |
| SL | 赤・破線 | ストップロス（想定損切位置） |

### 2. RSIチャート（下）

| 要素 | 説明 |
|------|------|
| RSI 線 | 紫色（0-100） |
| 30 ライン | 緑色・破線（Oversold） |
| 70 ライン | 赤色・破線（Overbought） |
| 背景色 | 0-30: 薄緑（買いシグナル可能性） |
| 背景色 | 70-100: 薄赤（売りシグナル可能性） |

### 3. タイトル情報

```
BTC/USDT 4H | RSI: 28.5 | Price: $42,500.00 | FLAT | Entry: $42,500.00 | R:R: 1:2.00
```

- **RSI 28.5** — Oversold領域（30以下）→ LONG シグナル候補
- **Price $42,500** — 現在価格
- **HOLDING** — ポジション中 / **FLAT** — ポジションなし
- **R:R: 1:2.00** — 1ドル損すると2ドル儲かる比率

## チャートの確認ポイント

### ✅ エントリー時の確認

```
1. ローソク足の位置を確認
   EMA(50) の上にあるか（LONG）/ 下にあるか（SHORT）

2. RSI の位置を確認
   - RSI < 30（Oversold）→ LONG シグナル候補
   - RSI > 70（Overbought）→ SHORT シグナル候補

3. SL・TP の位置を確認
   - SL は Entry の下（LONG）/ 上（SHORT）
   - TP は Entry の上（LONG）/ 下（SHORT）
   - R:R比が 1:2 以上あるか

4. タイトルで R:R比を確認
   - R:R: 1:2.00 以上が望ましい
```

### ⚠️ 想定と異なる表示の場合

| 現象 | 原因 | 対応 |
|------|------|------|
| SL・TP が表示されない | ログに SL/TP が記録されていない | ログを確認し、最新のエントリーがあるか確認 |
| EMA が表示されない | インジケーター計算エラー | ローソク足データが不足していないか確認 |
| チャートタイトルが "1H" | config.json の timeframe が古い | timeframe を "4h" に設定 |
| RSI 60 ライン（古い）が表示 | trade_chart.py が古いバージョン | ファイルを上書きして再度実行 |
